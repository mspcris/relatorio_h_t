Users API — Administração de usuários do Basic Auth (.htpasswd)

FastAPI + passlib.apache.HtpasswdFile com persistência em disco e endpoints REST.

Framework: FastAPI APIRouter (prefix="/users")
Estado: arquivo HTPASS = "/etc/nginx/.htpasswd"
Persistência: HtpasswdFile(..., autosave=True) (grava a cada operação)

Modelos (Pydantic)

  • CreateUser { email: EmailStr, password: str }
  • BlockReq { reason?: str } (campo não utilizado)
  • UnblockReq { new_password: str }

Endpoints efetivos

GET /users → retorna list[str] com os logins do .htpasswd.

Implementação: lê o arquivo, splitlines(), ignora comentários/linhas vazias, pega o trecho antes de :, ordena.

POST /users → cria/atualiza usuário.

Usa set_password(email, password). Cria se não existir; sobrescreve se existir. Retorno: {"ok": true}.

DELETE /users/{email} → remove usuário.

404 se não existe; caso contrário executa delete(email) e retorna {"ok": true}.

POST /users/{email}/block → bloqueia rotacionando a senha.

Gera nova senha aleatória com secrets.token_urlsafe(48) e grava. 404 se não existe. Retorno: {"ok": true}.

POST /users/{email}/unblock → redefine senha para new_password.

404 se não existe. Retorno: {"ok": true}.

Observações técnicas

  • Formato do arquivo: passlib grava em formato compatível com htpasswd; o hash depende da configuração padrão no ambiente.
  • Criação vs atualização: set_password sobrescreve silenciosamente (não há 409 por duplicidade).
  • Listagem: parsing manual assume linhas no padrão user:hash.
  • Autosave: sem transações; cada chamada persiste imediatamente. Em múltiplos processos, considere lock externo.
  • Validação: EmailStr apenas no POST /users; nos demais endpoints o e-mail é parâmetro de rota (precisa estar URL-encoded).
  • Segurança: não há autenticação/autorização na própria API. Publique atrás de um controle de acesso.
  • Uso de reason: recebido em /block, não é persistido nem logado.

Exemplos rápidos (cURL)

# listar
curl -s http://host/users

# criar/atualizar
curl -X POST http://host/users -H 'content-type: application/json' \
  -d '{"email":"alice@example.com","password":"S3nh@!"}'

# bloquear
curl -X POST http://host/users/alice@example.com/block \
  -H 'content-type: application/json' -d '{}'

# desbloquear
curl -X POST http://host/users/alice@example.com/unblock \
  -H 'content-type: application/json' \
  -d '{"new_password":"NovaS3nh@"}'

# remover
curl -X DELETE http://host/users/alice@example.com

Troque host pelo endereço do seu serviço. Garanta que o processo do Nginx lê o .htpasswd em /etc/nginx/.htpasswd.
