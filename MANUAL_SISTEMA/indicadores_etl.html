Python Requests Pandas IBGE SIDRA BCB OData/SGS Ipea OData / ipeadatapy CSV + JSON Site estático opcional

O que este código faz

Rodar: python .\indicadores_etl.py --ptax 2020-01-01 today --ipca 2020-01 thismonth --igpm 2020-01 thismonth --out .\json_consolidado

Orquestra um ETL de indicadores macro — PTAX (USD/BRL diário), IPCA (var. mensal do IBGE) e IGP-M (FGV via Ipeadata) — gerando CSV e JSON prontos para consumo em um diretório de saída. Opcionalmente, publica um site estático minimalista (HTML + CSS) para inspeção rápida dos dados. Entrega time-to-value alto com baixo atrito operacional.

Fontes de dados & fallback

  • PTAX (BCB/Olinda): CotacaoDolarPeriodo (compra/venda). Consolida por dia pegando o último registro do dia e calcula mid = (compra+venda)/2.
  • PTAX fallback (BCB/SGS): séries 1 (compra) e 10813 (venda) com parsing de valores no formato BR.
  • IPCA (SIDRA v3): agregado 1737, variável 63 (variação mensal, %), país (N1[all]), com normalização de vírgula decimal.
  • IGP-M:
    • Tentativa 1 via ipeadatapy (autodescoberta de série);
    • Tentativa 2 via OData (/ValoresSerie) usando códigos candidatos (IGP12_IGPMG12, etc.) ou o fornecido em --igpm-code.

Todas as chamadas HTTP usam retry exponencial com jitter e timeouts.

Entrada (CLI) & parsing de datas

  • --ptax (YYYY-MM-DD [YYYY-MM-DD] | today | yesterday)
  • --ipca (YYYY-MM [YYYY-MM] | thismonth | lastmonth)
  • --igpm (YYYY-MM [YYYY-MM] | thismonth | lastmonth); opcional --igpm-code com SERCODIGO.
  • --out diretório de saída (default: ./json_consolidado).
  • --site gera index.html + assets/style.css no --out. 
# Setup (Windows/PowerShell)
pip install -U requests pandas ipeadatapy

# Coleta 2020→atual e publica
python indicadores_etl.py --ptax 2020-01-01 today --ipca 2020-01 thismonth --igpm 2020-01 thismonth --out .\public
python indicadores_etl.py --site --out .\public

Pipeline (end-to-end)

  1. Parsings de datas/mês (suporta palavras-chave today/yesterday/thismonth/lastmonth).
  2. Coleta:
    • PTAX Olinda ▶️ fallback SGS.
    • IPCA SIDRA (v3, agregado 1737, var 63).
    • IGP-M ipeadatapy ▶️ fallback Ipea OData4.
  3. Normalização: datas ISO, campos padronizados, ordenação cronológica.
  4. Persistência: grava <name>.csv e <name>.json (orient="records").
  5. Site estático (opcional): injeta HTML com tabela simples e “últimos 6” registros por série.

Artefatos gerados

  • ptax_daily.csv / ptax_daily.json — campos: date, buy, sell, mid, source, as_of
  • ipca_monthly.csv / ipca_monthly.json — campos: ref_date, variable=ipca_mom, value, unit=% , source
  • igpm_monthly.csv / igpm_monthly.json — campos: ref_date, variable=igpm_mom, value, unit=% , source
  • index.html e assets/style.css (se --site informado).

Cada execução reporta wrote ... (N rows) para visibilidade operacional.

Regras de negócio & nuances

  • PTAX diário: consolida múltiplas cotações por dia mantendo apenas o último snapshot do dia.
  • IPCA/IGP-M: valores em % já normalizados (vírgula → ponto). A unidade é propagada no JSON.
  • IGP-M: fallback automático caso a biblioteca/endpoint primário não retorne estrutura esperada.
  • Resiliência: retries exponenciais com jitter para 5xx/timeout/conexão.

Estrutura do site estático

  • Estilo leve (assets/style.css), sem dependências externas.
  • Carrega ptax_daily.json, ipca_monthly.json e igpm_monthly.json e renderiza tabelas dos últimos 6 registros (via fetch).
  • Links de download para CSV/JSON em cada card.
