Node.js Express CORS REST Stub

O que este código faz?

Fora de uso, criado para testar CORS, mas, não evoluiu.

Visão executiva com linguagem direta e pragmática.

Resumo executivo

  • Sobe um microserviço Express (ESM) ouvindo na porta 3000 listen(3000).
  • Habilita parse de JSON no corpo das requisições via app.use(express.json()).
  • Configura CORS restrito ao origin https://teste-ia.camim.com.br via cors({ origin: [...] }).
  • Expõe POST /api/ia-kpi que recebe { query, page_ctx } e retorna uma resposta mockada (placeholder) com a pergunta.

Observação: o campo page_ctx é recebido mas não é utilizado; o comentário TODO indica onde plugar o seu LLM/serviço real.

Stack & Middleware

  • import express from 'express' — framework HTTP.
  • import cors from 'cors' — controle de origens.
  • app.use(express.json()) — body parser para JSON.
  • app.use(cors({ origin: ['https://teste-ia.camim.com.br'] })) — libera browser calls apenas desse domínio.

Endpoint (mock)

POST /api/ia-kpi
Body: { "query": "string", "page_ctx": any }
Resp: { "answer": "Ok. Pergunta: \"<query>\". Exemplo de resposta baseada nos KPIs." }

Responsável por orquestrar a resposta dos KPIs (ainda stub).

Exemplo de consumo (cURL)

curl -s -X POST http://localhost:3000/api/ia-kpi \
  -H "Content-Type: application/json" \
  -d '{"query":"qual o IPCA do mês?","page_ctx":{"empresa":"XPTO"}}'

Resposta

{
  "answer": "Ok. Pergunta: \"qual o IPCA do mês?\". Exemplo de resposta baseada nos KPIs."
}

CORS — comportamento esperado

  • Permitido: chamadas do browser vindas de https://teste-ia.camim.com.br.
  • Bloqueado (preflight): outros domínios (ex.: localhost:5173) — ajuste o array de origin se necessário.
  • Via cURL/Postman: CORS não se aplica (só browsers checam CORS).

Boas práticas para produção (hardening)

  • Porta por ambiente: app.listen(process.env.PORT || 3000).
  • Validação de payload: garantir query presente, tamanho máx., sanitização.
  • Auth & rate limit: proteção do endpoint se exposto publicamente.
  • CORS dinâmico: liberar localhost em dev, domínio oficial em prod.
  • Observabilidade: logs estruturados, tracing e métricas.
  • Erro handling: try/catch no handler e respostas com códigos HTTP adequados.
