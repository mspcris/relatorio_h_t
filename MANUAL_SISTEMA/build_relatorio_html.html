Script de Relatório — Consolidação Trello + Harvest

Gera uma página HTML com tabelas filtráveis a partir dos exports mais recentes do Trello e Harvest, com visão de horas por cliente, projeto e usuário.

Stack: Python 3 + pandas + string.Template
Entrada Trello: última pasta em export_trello/export_YYYYMMDD_HHMMSS/
Entrada Harvest: CSV mais recente em export_harvest/
Saída: relatorio/relatorio_YYYYMMDD_HHMMSS.html e trello_harvest.html

O que o script faz (end-to-end)

  1. Descoberta de fontes:
    • Trello: encontra a subpasta mais recente em export_trello/ que contenha cards.csv (e opcionalmente lists.csv).
    • Harvest: escolhe o CSV mais novo em export_harvest/.
  2. Leitura resiliente de CSV: tenta utf-8; se falhar, cai para latin-1.
  3. Pipeline Trello:
    • Se houver lists.csv, mapeia idList → list (nome da lista).
    • Ordena os cards por recência (usa a primeira coluna de data disponível: updated, dateLastActivity, due, start, created/created_at).
    • Recorta colunas-chave e renomeia: name → card, adiciona link clicável em url.
  4. Pipeline Harvest:
    • Normaliza nomes de colunas para um padrão (date, project, client, task, hours, notes).
    • Detecta a coluna de usuário (pick_username_column) e padroniza para user.name.
    • Converte hours para número e cria três pivôs: por cliente, por projeto e por usuário (somatório de horas).
    • Formata hours em HH:MM tanto na tabela principal quanto nos pivôs.
  5. HTML & UX:
    • Renderiza tabelas em HTML com um input de filtro (texto) por tabela.
    • Preenche placeholders do template ($now, $trello_hint, $harvest_hint e blocos de tabelas).
  6. Persistência de saída:
    • Gera relatorio/relatorio_YYYYMMDD_HHMMSS.html (datado).
    • Sobrescreve trello_harvest.html na raiz do projeto como atalho para o último relatório.
    • Imprime no stdout: OK: caminhos gerados.

Assunções de diretório

.
├─ export_trello/
│  ├─ export_20250110_093000/
│  │  ├─ cards.csv
│  │  └─ lists.csv
│  └─ export_20241201_101500/
│     └─ cards.csv
├─ export_harvest/
│  ├─ harvest_2025-01.csv
│  └─ harvest_2025-02.csv   <-- escolhido (mais recente)
├─ relatorio/
│  └─ relatorio_YYYYMMDD_HHMMSS.html   <-- novo
└─ trello_harvest.html                  <-- sempre sobrescrito

A “última” pasta/arquivo é definida pelo mtime (data de modificação no sistema de arquivos).

Como rodar

# dentro da raiz do projeto (onde existem export_trello/ e export_harvest/)
python3 relatorio_trello_harvest.py \
  --root . \
  --outdir relatorio

# parâmetros
--root    caminho da raiz do projeto (default: .)
--outdir  pasta de saída dos relatórios (default: relatorio)

Dependências-chave: pandas.

Principais funções utilitárias

  • read_csv_safe(path): abre CSV em UTF-8; se falhar, tenta Latin-1.
  • latest_harvest_csv(root): retorna o CSV mais recente em export_harvest/.
  • latest_trello_dir(root): retorna a subpasta mais recente em export_trello/ com cards.csv.
  • sort_by_recent(df, cols): converte colunas de data para datetime (UTC) e ordena por recência.
  • pick_username_column(df): tenta identificar a coluna com o nome do usuário e padroniza como user.name.
  • fmt_hours_hhmm(x): converte horas decimais para HH:MM.
  • df_to_html_table(df,...): monta tabela HTML com filtro embutido; permite colunas com HTML seguro.

Template HTML produzido

  • Inclui cabeçalho, menu lateral (referências a ../js/*.js e ../css/style.css), e duas abas: Harvest (entradas + pivôs) e Trello (cards).
  • Cada tabela acompanha um campo “Filtrar nesta tabela…”.
  • Placeholders resolvidos dinamicamente:
    • $now — timestamp de geração
    • $trello_hint — nome da pasta do export Trello usado
    • $harvest_hint — nome do CSV do Harvest usado

Riscos, bordas e boas práticas

  • Layout de CSV variável: o script tenta mapear nomes de colunas por heurística. Caso os exports mudem, valide o renomeador.
  • Datas heterogêneas: ordenação por recência usa a primeira coluna de data disponível; garanta consistência de fuso/UTC se comparar com outros relatórios.
  • UTF-8 vs Latin-1: o fallback cobre casos comuns, mas padronize a codificação dos exports para reduzir ruído.
  • Sobrescrita do índice: trello_harvest.html é sempre substituído pelo último build.
  • Links Trello: a coluna url é renderizada como <a target="_blank">, garantindo navegação direta para o card.

Deliverables (saídas geradas)

Arquivo Descrição
relatorio/relatorio_YYYYMMDD_HHMMSS.html Relatório datado para versionamento/auditoria.
trello_harvest.html Atalho “sempre atualizado” para o último relatório gerado.

Checklist de validação pós-build

  • Badges do cabeçalho mostram corretamente Trello: nome da pasta e Harvest: nome do arquivo?
  • Tabela “Entradas - Harvest” exibe horas em HH:MM (ex.: 01:30)?
  • Pivôs (“por Cliente/Projeto/Usuário”) somam e ordenam horas conforme esperado?
  • Cards do Trello vêm ordenados por atividade/datas mais recentes?
  • Campo “Filtrar nesta tabela…” oculta/exibe linhas conforme o texto?

Qualquer ajuste fino (ex.: colunas extras, novos cálculos) pode ser plugado antes da renderização no objeto harvest_display e nos pivots.
