EXPORT_GOVERNANCA.PY

Parâmetros

--from YYYY-MM
Início do intervalo. Default: 2020-01.
--to YYYY-MM
Fim exclusivo do intervalo. Default: início do mês corrente.
--only-month YYYY-MM
Executa só um mês específico.
--postos ANX
Subset de postos. Default: ANXYBRPCDGIMJ.
--force
Sobrescreve qualquer CSV existente em todo o intervalo.
--outubro
Regera o mês imediatamente anterior ao atual: apaga os CSVs desse mês e força sobrescrita dele e do mês corrente.
--no-validate
Não valida conteúdo dos .sql.
--dry-run
Executa sem gravar CSV/JSON.
--copy-templates
Copia templates/ para TARGET_TEMPLATES_DIR (env ou public/).

Observações

  • Priorize --only-month quando quiser isolar um mês.
  • --force ignora existência de arquivos. --outubro afeta apenas mês anterior + mês corrente.
  • SQLs podem usar parâmetros :ini e :fim ou ? posicionais.

Playbook de execução

  1. Rodada padrão mês a mês até o mês atual

    python export_governanca.py

  2. Regenerar mês anterior + mês corrente

    python export_governanca.py --outubro

  3. Reprocessar um mês específico

    python export_governanca.py --only-month 2024-08

  4. Forçar sobrescrever tudo para um intervalo

    python export_governanca.py --from 2024-01 --to 2024-12 --force

  5. Apenas validar pipeline sem gravar

    python export_governanca.py --from 2024-05 --to 2024-09 --dry-run

  6. Rodar para postos selecionados

    python export_governanca.py --postos ANX

  7. Copiar templates do front após a geração

    python export_governanca.py --copy-templates
