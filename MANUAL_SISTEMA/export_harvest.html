Python Requests pandas dotenv Harvest API

O que este código faz

Orquestra um export incremental rolling dos time entries do Harvest referentes aos últimos 7 dias corridos (inclui hoje), materializando um CSV novo por execução em export_harvest/. É plug-and-play para alimentar dashboards/ETLs.

Pré-requisitos (variáveis de ambiente)

  • HARVEST_TOKEN (obrigatório) — token OAuth/Personal do Harvest.
  • HARVEST_ACCOUNT_ID (obrigatório) — ID da conta.
  • USER_AGENT (opcional) — default CamimReports/1.0.
# .env (na raiz)
HARVEST_TOKEN=xxxxxxxxxxxxxxxx
HARVEST_ACCOUNT_ID=123456
USER_AGENT=CamimReports/1.0

Sem HARVEST_TOKEN e HARVEST_ACCOUNT_ID o script aborta com SystemExit.

Fluxo ponta a ponta

  1. Carrega env: load_dotenv() e injeta cabeçalhos da API (Bearer + Account ID + User-Agent).
  2. Janela de extração: calcula from = hoje − 6 dias e to = hoje no formato YYYY-MM-DD.
  3. Paginação resiliente: fetch_time_entries() chama GET /v2/time_entries com per_page=2000, iterando page até total_pages. Usa timeout=60 e raise_for_status().
  4. Normalização tabular: pandas.json_normalize(). Se houver dados, filtra colunas-chave (user, project, task, client, horas, datas, flags) e ordena por spent_date, user.name, project.name.
  5. Persistência idempotente por run: garante export_harvest/, compõe nome único com harvest_time_entries_{from}_{to}_{timestamp}.csv e grava via to_csv() (UTF-8 padrão).
  6. Retorno de status: devolve string do tipo OK: N linhas > caminho/arquivo.csv.

Outputs & naming convention

  • Diretório de saída: export_harvest/
  • Arquivo: harvest_time_entries_<YYYY-MM-DD_from>_<YYYY-MM-DD_to>_<YYYYMMDD_HHMMSS>.csv
  • Cada execução gera um arquivo novo (nunca sobrescreve).

Se a janela não retornar registros, o CSV pode sair vazio (sem colunas). É comportamento padrão do pandas com DataFrame vazio.

Contrato de dados (quando há colunas)

  • id, spent_date, hours, notes
  • user.id, user.name
  • project.id, project.name
  • task.id, task.name
  • client.id, client.name
  • billable, is_locked, created_at, updated_at

O subconjunto efetivo respeita apenas as colunas realmente retornadas pela API nesse período.

Como executar

# 1) Crie o .env com as credenciais
# 2) Instale dependências
pip install python-dotenv requests pandas

# 3) Rode o script
python export_harvest_last7.py   # ou o próprio arquivo fornecido

# Saída esperada no STDOUT:
OK: 128 linhas > export_harvest/harvest_time_entries_2025-10-27_2025-11-02_20251102_101530.csv

Notas de operação & riscos

  • Rate limits: respeite as cotas do Harvest; o script pagina com 2000 por página para reduzir chamadas.
  • Observabilidade: falhas HTTP estouram via raise_for_status(); trate onde integrar.
  • Privacidade: os CSVs podem conter nomes/descrições (notes); versionar com critério.
  • Timezone/Data: a API usa datas em formato ISO; valide fuso se cruzar com BI.
