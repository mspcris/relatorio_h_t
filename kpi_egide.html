Como ler esta página
A Égide Saúde & Benefícios é um marketplace de saúde: clínicas, médicos e farmácias se cadastram, e clientes marcam consulta, exame ou compram remédio pelo site/app. É da CAMIM, mas tem banco de dados separado.
  • Consulta — cliente vai num médico de uma especialidade.
  • Exame — cliente vai fazer um exame (sem médico fixo).
  • Particular — cliente pagou do próprio bolso (cartão, PIX, boleto).
  • Convênio — plano de saúde (Unimed, Bradesco…) cobre. Quem paga é o convênio, não o cliente.
⚠️ Pra não confundir: Receita = dinheiro que entrou. Consulta paga = cliente já quitou o valor (não só agendou). Agendamento pode ser cancelado depois — por isso os números de agendadas e pagas são diferentes.
Dados a partir de janeiro/2023. Mês em andamento aparece com PARCIAL.

3. Particular vs Convênio

Qual a diferença?
  • Particular — cliente pagou do bolso. Égide recebe na hora.
  • Convênio — plano de saúde cobre. Égide cobra depois, do convênio.
Para saber se uma consulta é de convênio, checamos se o cliente tinha convênio ativo na data (tabela customer_insurances).
📊 O que você vai ver: a pizza soma o período filtrado — a fatia de convênio costuma ser pequena perto de particular.

4. Arrecadação por contrato (Particular)

O que é "arrecadação por contrato"?
É o dinheiro que a Égide recebe direto do cliente, sem convênio no meio. Vem de dois canais:
  • Clínica — cliente pagou uma consulta ou exame. Isso inclui:
    • pedidos do marketplace com orderType=clinic; e
    • consultas pagas direto na tabela doctorappointments sem passar pelo fluxo de orders (fluxo alternativo; é receita real da Égide).
  • Farmácia — cliente comprou remédio ou produto pelo app.
Se aparecer Outros, é porque surgiu um orderType novo além de clinic/pharmacy.
💡 Leitura: cada barra é um mês, empilhada pelos canais. Dá pra ver se a farmácia cresceu, se a clínica puxou, etc.
