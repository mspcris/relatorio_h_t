Explicação do SELECT (Snapshot mensal) 1 linha por médico • Último dia do mês • Prioriza PJ

Este SELECT foi desenhado para gerar um snapshot mensal do histórico de especialidades: para cada mês, ele considera somente os registros do último dia do mês e devolve apenas 1 linha por médico (sem duplicar por múltiplas especialidades), priorizando registros em que PessoaJuridica = 1.

Objetivo prático

Ter uma “foto do mês” do seu quadro de médicos no fechamento, para medir KPIs como % de PessoaJuridica preenchido mês a mês e montar análises (ex.: Pareto).

Resultado esperado

Em cada DataFechamentoMes, cada idmedico aparece uma única vez, mesmo que ele tenha várias especialidades. No mês seguinte, ele aparece novamente (novo snapshot).

1 CTE base: prepara e “limpa” os dados

Aqui o SELECT lê o histórico (Cad_EspecialidadeHistorico) e faz o join com o cadastro do médico (cad_medico) para trazer nome, CRM e indicadores (ex.: Pessoa Jurídica).

  • Cria DataDia: pega somente a parte da data (sem hora) de DataHoraInclusao.
  • Cria DataFechamentoMes: calcula o último dia do mês com EOMONTH(DataHoraInclusao) (28/29/30/31).
  • Aplica filtros de qualidade: remove registros desativados (eh.Desativado = 0), nomes internos/fake e especialidades fora do escopo.

Resultado desta etapa: uma base consistente para análise, mas ainda pode conter duplicidades (ex.: 2 especialidades para o mesmo médico).

2 CTE fechamento: mantém só o último dia de cada mês

Nesta etapa, o SELECT filtra a base para ficar apenas com registros do dia de fechamento do mês:

  • Regra: DataDia = DataFechamentoMes
  • Exemplo: se o mês tem 30 dias, ficam apenas linhas do dia 30; se tem 31, apenas do dia 31.
Leitura simples: “Eu só quero olhar o quadro de médicos no fechamento do mês, não o mês inteiro.”

3 CTE dedup: remove duplicidade por médico (prioriza PJ=1)

Mesmo no fechamento, um médico pode ter várias linhas (por múltiplas especialidades ou duplicação no histórico). Para garantir 1 linha por médico por mês, o SELECT usa ROW_NUMBER().

  • Particiona por (DataFechamentoMes, idmedico): “para cada mês e para cada médico, numere as linhas”.
  • Define qual linha “ganha” (fica como rn = 1):
    1. Prioridade: primeiro quem tem PessoaJuridica = 1
    2. Desempate 1: mais recente dentro do dia (DataHoraInclusao DESC)
    3. Desempate 2: especialidade em ordem alfabética (só para não ficar aleatório)

Resultado desta etapa: para cada mês (fechamento), cada médico aparece uma única vez, com preferência por registros PJ.

4 SELECT final: entrega o snapshot pronto para KPI

No final, o SELECT escolhe apenas a linha “vencedora” de cada médico: WHERE rn = 1 e ainda remove médicos temporários: AND Temporario = 0.

  • certificadodigital vira um indicador (0/1) baseado em CertificadoPFX.
  • rqe vira um indicador (0/1) baseado em NumeroRQE.
  • Ordena por DataFechamentoMes e nome do médico para leitura/extração.
Resultado final: “Para cada mês, no último dia, me traga a lista de médicos (sem duplicar), escolhendo a melhor linha por médico, e ignorando temporários.” 

-- SQL_CTRLQ_RELATORIO.SQL
WITH base AS (
  SELECT
    eh.DataHoraInclusao,
    CONVERT(date, eh.DataHoraInclusao) AS DataDia,
    EOMONTH(eh.DataHoraInclusao)       AS DataFechamentoMes,

    m.idmedico,
    m.nome,
    m.crm,
    eh.Especialidade,

    m.CertificadoPFX,
    eh.PermitirAgendamentoquenuncaconsultou,
    m.PessoaJuridica,
    eh.NumeroRQE,
    eh.temporario
  FROM Cad_EspecialidadeHistorico eh
  LEFT JOIN cad_medico m ON m.idmedico = eh.idmedico
  WHERE eh.Desativado = 0
    AND m.nome NOT LIKE '%sede%'
    AND m.nome NOT LIKE '%agendamento%'
    AND m.nome NOT LIKE '%teste%'
    AND m.nome NOT LIKE '%fake%'
    AND eh.especialidade NOT IN (
      'enfermeiro','eletrocardiograma','espirometria','biologista','mamografia',
      'tecnico em radiologia','HOLTER 24 HRS','mapa'
    )
),
fechamento AS (
  SELECT *
  FROM base
  WHERE DataDia = DataFechamentoMes   -- só o último dia do mês (30/31/28/29)
),
dedup AS (
  SELECT
    *,
    ROW_NUMBER() OVER (
      PARTITION BY DataFechamentoMes, idmedico
      ORDER BY
        CASE WHEN PessoaJuridica = 1 THEN 0 ELSE 1 END, -- prioriza PJ=1
        DataHoraInclusao DESC,                          -- desempate: mais recente no dia
        Especialidade ASC                               -- desempate final determinístico
    ) AS rn
  FROM fechamento
)
SELECT
  DataFechamentoMes,
  DataHoraInclusao,
  nome AS medico,
  crm,
  especialidade,
  temporario,
  IIF(CertificadoPFX IS NULL, 0, 1) AS certificadodigital,
  permitiragendamentoquenuncaconsultou,
  pessoajuridica,
  IIF(NumeroRQE IS NULL, 0, 1) AS rqe
FROM dedup
WHERE rn = 1 and Temporario = 0
ORDER BY DataFechamentoMes, medico;
-- END SQL_CTRLQ_RELATORIO.SQL
