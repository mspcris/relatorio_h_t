    1) Total Beneficiários se refere a clientes CAMIM, CAMIM Premium e Liberty.

    2) Ticket Médio Real PF é o valor do tícket consierando as mensalidades pagas, ou seja, se pago antes do vencimento, tem desconto, etc.

    Manual Técnico — Origem dos Dados

    1. Visão Geral

    Este painel consolida dados de matrículas PF, matrículas PJ, vidas e beneficiários, provenientes diretamente das views e tabelas do banco operacional da CAMIM. As cargas são executadas via export_cad_cliente_incremental.py, que gera os arquivos JSON consumidos por este dashboard.

    3. Lógica de Cálculo (Resumo)
    • Matrículas PF/PJ: somatório por tipo de contrato, sendo considerado planos PJ's matrículas com mensalidade de valor mínimo de R$ 48, para evitar matrículas feitas para funcionários, reserva de vagas, permuta, etc. Matrículas Taxi são consideradas matrículas normais CAMIM.
    • Tickets PF/PJ: média ponderada por matrícula. O valor real considera o valor pago na última mensalidade que pode não ter sido no mês vigente, logo, é uma visão, apenas. Para visualizar dados reais sobre ticket médio, por favor, abra o kpi de mensalidades. O valor previsto considera o valor da mensalidade do cliente. Desta forma é posível considerar ganhos com pagamentos em atraso, mas, da mesma forma clientes que pagam em dia e possuem desconto ponderam o valor.
    • Vidas: somatório por vidas em matrículas ativas, não importando se são CAMIM, Correios, etc.
    • Beneficiários: somatório por faixa etária (PF e PJ separadamente). Sendo considerado PF somente planos CAMIM, CAMIM PREMIUM e Liberty.
    • Consolidado Geral: união de PF + PJ por posto.
    4. SQL Utilizado (Exemplo)

    A consulta abaixo demonstra como os dados de matrículas PF/PJ são originados:

    Select
e.codigo AS posto,
vwc.tipo,
count(*) as matriculas, avg(vwc.Valormensalidadeultimopagamento) as ticket_medio_real, 
avg(vwc.[valor mensalidade]) as ticket_medio_previsto, sum(vwc.Valormensalidadeultimopagamento) as totalultimamensalidade
from 
vw_Cad_Cliente vwc
join sis_empresa emp on emp.idendereco = vwc.idendereco
join Cad_Endereco e on emp.idEndereco = e.idEndereco
Where (vwc.Desativado = 0)
 and (vwc.[Situação] = 'adimplente')
 and vwc.idEndereco = emp.idEndereco
 AND (vwc.CanceladoANS = 0)
 and vwc.Convênio = 0
 and vwc.[Valor mensalidade] > 48
 group by e.codigo, vwc.tipo

    *Este SQL é apenas um exemplo demonstrativo. Ajuste conforme necessário.

    5. SQL Utilizado (Exemplo)

    A consulta abaixo demonstra como os dados de matrículas beneficiários PJ são originados:

    SELECT
              e.codigo AS posto,
    CASE
        WHEN cp.idade BETWEEN  0 AND 18 THEN '0 a 18'
        WHEN cp.idade BETWEEN 19 AND 23 THEN '19 a 23'
        WHEN cp.idade BETWEEN 24 AND 28 THEN '24 a 28'
        WHEN cp.idade BETWEEN 29 AND 33 THEN '29 a 33'
        WHEN cp.idade BETWEEN 34 AND 38 THEN '34 a 38'
        WHEN cp.idade BETWEEN 39 AND 43 THEN '39 a 43'
        WHEN cp.idade BETWEEN 44 AND 48 THEN '44 a 48'
        WHEN cp.idade BETWEEN 49 AND 53 THEN '49 a 53'
        WHEN cp.idade BETWEEN 54 AND 58 THEN '54 a 58'
        WHEN cp.idade >= 59            THEN '59+'
        ELSE 'Sem idade'
    END AS faixa_etaria,
    COUNT(*) AS total
FROM vw_Cad_PacienteView cp
LEFT JOIN sis_empresa emp ON emp.idendereco = cp.idendereco
join cad_endereco e on e.idendereco = emp.idendereco
left join vw_cad_cliente cc on cc.idcliente = cp.idcliente
WHERE cp.desativado = 0
  AND cp.idade IS NOT NULL
  AND cp.[Situação] = 'Adimplente'
  AND cp.tipo_FJ = 'J'
  AND cp.CanceladoANS = 0
  AND cp.tipo = 'dependente'
  AND cp.matricula <> emp.MatriculaFuncionario 
  AND cc.[Valor mensalidade] > 48
GROUP BY
    e.codigo,
    CASE
        WHEN cp.idade BETWEEN  0 AND 18 THEN '0 a 18'
        WHEN cp.idade BETWEEN 19 AND 23 THEN '19 a 23'
        WHEN cp.idade BETWEEN 24 AND 28 THEN '24 a 28'
        WHEN cp.idade BETWEEN 29 AND 33 THEN '29 a 33'
        WHEN cp.idade BETWEEN 34 AND 38 THEN '34 a 38'
        WHEN cp.idade BETWEEN 39 AND 43 THEN '39 a 43'
        WHEN cp.idade BETWEEN 44 AND 48 THEN '44 a 48'
        WHEN cp.idade BETWEEN 49 AND 53 THEN '49 a 53'
        WHEN cp.idade BETWEEN 54 AND 58 THEN '54 a 58'
        WHEN cp.idade >= 59            THEN '59+'
        ELSE 'Sem idade'
    END
ORDER BY e.codigo, faixa_etaria;

    *Este SQL é apenas um exemplo demonstrativo. Ajuste conforme necessário.
